# Re-entry Summary

## Purpose

Make the next session able to continue without rediscovering context.

## Include

1. Current state.
2. Last completed action.
3. Next recommended action.
4. Important decisions.
5. Open issues.
6. Files changed.
7. Verification run.
8. What not to redo.
9. Links to canon/index/context pack if present.

## Rules

- Prefer concise state over full history.
- Mark stale or superseded information.
- Do not include secrets.
